Descartes

Styles that think with JavaScript

An experimental library for reactive, computable CSS with DOM access

var reset = {"margin": 0, "padding": 0, "box-sizing": "border-box"};
var rand_angle = function() { return Math.round(Math.random() * (180) - 90); };
var rand_rgba = function() {
  return "rgba("+[255,255,255].map(x => {
    return Math.round(Math.random() * x);
  }).join()+", 1)"
};

new Descartes({
  "html": {
    "_mixins": reset, // Reuse CSS rules with mixins
    "body": { // Nest rules just like in Sass and Less
      "_mixins": reset, // Look! Reused CSS rules
      "_listeners": [[window, "click"]], // Bind event listeners to rules...
      "background": function() {
        return 'linear-gradient('
          + rand_angle().toString() + 'deg,'
          + rand_rgba() + ','
          + rand_rgba() + ')'
      } // ...to set dynamic property values when any listened events fire!
    }
  }
})

Style Tree

Write your styles in a JSON object

I've taken some ideas from CSS preprocessors like Less and Sass and wondered what it would be like to write CSS as JSON. It gives you all the advantages of a typical preprocessor and some. It means you can do real programming to set your styles, get access to the DOM, and even bind listeners.

CSS 
html {
  margin: 0;
  padding: 0;
}
html body {
  margin: 0;
  padding: 0;
}
html body section {
  max-width: 800px;
}
Sass 
html {
  margin: 0;
  padding: 0;
  body {
  margin: 0;
  padding: 0;
    section {
      max-width: 800px;
    }
  }
}
Descartes + JavaScript 
new Descartes({
  "html" {
    "margin": 0,
    "padding": 0,
    "body" {
    "margin": 0,
    "padding": 0,
      "section" {
        "max-width": "800px"
      }
    }
  }
})

 

Explicit

Makes styles easier to understand

Cascading in stylesheets can be pretty awesome, but it gets in the way once your applications need to scale. Descartes insists that you be explicit about your styles, putting emphasis on styles that are deeply nested and using mixins to reuse styles that typically cascade.

Dynamic Properties

Pass functions as CSS values

You can set style properties as JavaScript functions that are accessed every time the Style Tree is rendered. This gives you access to the DOM, variables, and any of ther JavaScript you want!

 
 

Listeners

Bind events that updated your styles

You can also attach listeners to different DOM events that render a specific set of rules. That makes things like parallax, window based positioning, and even interactivity a breeze.

Mixins

Reuse your styles and functions

Reuse rules that you use for multiple elements by creating mixins. They get reapplied to each element you specify it for, letting you apply things like consistent styles, listeners, and even other mixins.
